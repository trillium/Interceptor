## Important conclusion

The flow is partially working live:
- navigation worked
- network capture worked
- page parsing worked enough to get title/organizer/date/attendee summary
- screen evidence was captured

But the full extraction is **not yet correct** for:
- linked post text
- poster name
- author follower count
- engagement counts
- details tab content
- ISO timestamps
- thumbnail quality

## Next implementation targets

1. Restrict post-card extraction to the event post container, not any large visible card or messaging overlay.
2. Prefer event post author/follower block over host/speaker profile blocks.
3. Extract visible engagement counts directly from the event post section as a fallback.
4. Improve Details-tab targeting and content scoping.
5. Improve network-response scoring for actual event metadata endpoints so start/end ISO timestamps are populated.
6. Exclude placeholder/tracking images from thumbnail selection.
